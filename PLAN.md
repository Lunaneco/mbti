# 画像アセット仕分け計画: 星詠み16タイプ診断ゲーム

## 1. 方針

完成版品質では、**文字・ボタン状態・進捗・演出はできるだけコードで表現**し、画像生成は「世界観の説得力を作る大きな絵」に集中する。

重要ルール:

- 生成画像には原則として文字を入れない。日本語文字は崩れやすく、差し替えも難しいため、タイトル・質問・結果名・ボタン文言はコードで重ねる。
- 既存画像は「完成素材」兼「方向性の基準」として使う。ただし `title.png`、`questionframe.png`、`mbtishindan.jpg` は文字が焼き込まれているため、実装用には文字なし版を用意する。
- 画面比率は既存に合わせる。タイトル/質問画面は `1086 x 1448`、結果カードは `1024 x 1536` または高解像度版 `2048 x 3072` を推奨。

## 2. 既存素材の使い分け

| ファイル | 使い方 | 画像生成の要否 |

| `questionframe.png` | 質問画面の完成イメージ参考 | 実装用に文字なし版を生成/編集したい |

| `assets.png` | ボタン・装飾・アイコンの素材参考、切り出し元 | 生成ではなく切り出し/コード再現でよい |
| `必要アセット素材.png` | 全体UIの仕様サンプル | 参考用。直接使うより個別パーツ化する |

## 4. コードで表現できるもの

画像生成しなくてよいもの:

- タイトル文字、サブタイトル、注意書き
- 質問文、選択肢A/B、Q番号
- `戻る`、`次へ`、`診断をはじめる` などのボタン文字
- ボタンの通常/ホバー/選択/非選択/無効状態
- 進捗バー、ステップ宝石、回答済みの発光
- 星粒、きらめき、ゆっくり回る光、カードめくり
- 結果タイプ名、カード名、称号、引用文
- 強み/弱み/相性バッジ
- レア度ラベル、`NEW`、`おすすめ`、`人気`
- 軸バランスゲージ、パーセンテージ表示
- スマホ/PCのレスポンシブ配置
- 結果カードのPNG保存処理

コード表現の推奨:

- 金枠ボタンはCSSの `border-image`、背景グラデーション、`box-shadow` で作る。
- 星や宝石の進捗はCSSまたはSVGで作る。
- 装飾ラインは `assets.png` から切り出した透明PNGを左右反転・回転して再利用する。
- 文字はWebフォントで表現する。英字は `Cinzel`、日本語見出しは `Shippori Mincho`、本文は `Noto Sans JP` が合う。

## 5. 最終的に生成すべき画像数

完成版品質の推奨セット:

- 画面背景: 4点
- 結果カード共通パーツ: 4点
- 16タイプ結果キャラクター/カード絵: 16点
- 案内役/吹き出し: 7点
- 追加演出/OGP: 4点

合計: **35点**

最小限に削るなら:

- タイトル文字なし背景
- 質問文字なしフレーム
- 結果カード共通フレーム
- 16タイプ結果絵

合計: **19点**

既存素材を最大活用するなら、`mbtishindancardura.png` はそのままカード裏/分析中に使用し、`assets.png` はボタンや装飾を切り出す。新規生成の中心は **16タイプの結果絵** と **文字なし画面背景** に絞る。
